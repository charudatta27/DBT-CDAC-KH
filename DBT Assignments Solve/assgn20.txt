1) Which of these views are updateable (will allow DML operations)?
#1 Create View Dailyorders
as Select Distinct cnum, snum, onum, odate from Orders;
#2 Create View Custotals
as Select cname, Sum (amt) Sum_Amt from Orders, Customers
where Orders.cnum=Customers.cnum
Group by cname;
#3 Create view Thirdorders
as Select * from Dailyorders where
odate=’1990-10-03’;
#4 Create view Nullcities
as Select snum, sname, city
from Salespeople
where city is NULL
OR sname BETWEEN ‘A’ and ‘MZ’;
-->

In general, a view is NOT updateable if it contains DISTINCT, GROUP BY, Aggregate functions (like SUM), or references multiple tables in certain ways.

#1 (Dailyorders): Not Updateable. It uses the DISTINCT keyword.

#2 (Custotals): Not Updateable. It uses the SUM() aggregate function and a GROUP BY clause.

#3 (Thirdorders): Not Updateable. Since it is built on top of Dailyorders (which is already non-updateable due to DISTINCT), the restriction carries over.

#4 (Nullcities): Updateable. It selects simple columns from a single table (Salespeople) without any prohibited keywords like DISTINCT or GROUP BY









2) Create a view of the Salespeople table called Commissions. This view will include
only the snum and comm fields. Through this view, someone could enter or change
commissions, but only to values between .10 and .20.

-->


CREATE VIEW Commissions AS
SELECT Snum, comm
FROM SALESPERSON
WHERE comm BETWEEN 0.10 AND 0.20
WITH CHECK OPTION;
